Localization

In the Yellow code view you can use the following syntax to define a localizable text: @{{key, arg0, arg1, arg2, ...}}

Parameter key is the key of the localized message (without quotes), the additional arguments are the expressions which will be evaluated by Yellow code and passed to the translation function. Any valid expression which can be used in data-ylcBind mapping or in {{...}} is also valid here.

Example: @{{key.of.localizable.message, 55, modelObject.field1 + modelObject.field2, 'cool'}}

Yellow code itself doesn't define the translation function. For the localizable strings to work, the controller has to define a function called translate, which takes the message key as its first parameter and the values of the expressions (after evaluation) as its additional parameters. Typically, the translate function will be placed to the global mixing so that it's available to all views.

The example below shows a "poor man's" translate function implementation. For production consider using some mature library such as intl-messageformat. In essence it is very straightforward to integrate any localization library with Yellow code: simply define a global mixin with the translate method which will pass its arguments to your chosen localization library and return the result.
