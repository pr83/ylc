Example: Nested views

The following example demonstrates that a Yellow code view can be nested inside another Yellow code view and each view can have its own model and controller.

The first view is a simple color picker. It is actually wrapped inside a simple jQuery plugin called colorWidget. This plugin transforms a DIV it is applied to into a colored rectangle which, when clicked, opens a color picker and when the user selects a color in the color picker, the color of the rectangle is changed and the picker is closed. The plugin further defines a jQuery-style getter/setter to read/change the current color programmatically:

  • $(...).colorWidget("color") to read the current color,
  • $(...).colorWidget("color", "green") to change the current color.

Since Yellow code understands the jQuery getter/setter convention, the color of the rectangle can be bound to the model like this: <div data-ylcBind="colorWidget.color: color"> This binding is defined by the outer Yellow code view. The outer view simply defines a couple of rectangles in various colors. The colorful rectangles are DIVs onto which the colorWidget has been applied as soon as the DIVs became initialized.
